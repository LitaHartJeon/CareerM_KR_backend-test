# MOFIN 2026 1Q BACKEND TEST

## 배경
지원자는 다른 개발자가 사용할 암호화폐 시세 서버를 개발하게 됩니다. 서버는 바이낸스에서 조회한 시세 데이터를 다른 어플리케이션에 다양한 API로 제공합니다.

## 절차
1. 지원자는 Collaborator로 초대된 리포지터리를 포크합니다.
2. 포크한 리포지터리에서 지원자는 테스트를 수행합니다.
3. 테스트 완료 시 지원자는 포크한 리포지터리의 URL과 완료 여부를 이메일로 회신합니다.
4. 평가자는 지원자가 작성한 README 설명대로 평가합니다.
5. 평가자는 평가 완료 후 24시간 이내에 지원자에게 평가 결과를 통보합니다.

## 조건
- 본 리포지터리에 Collaborator로 초대된 시점으로부터 72시간 이내의 커밋까지 인정됩니다.
    - 보다 이른 시간에 완료한 경우 가점에 반영됩니다.
- 본 리포지터리를 포크하여 진행해야 합니다.
- JavaScript로 개발한 어플리케이션이어야 합니다.
    - 그외 기술에 대한 선택은 자유입니다.
- 시세 데이터는 매수가, 매수량, 매도가, 매도량, 시세 시점을 포함해야 합니다.
- 지원자가 README에 제공한 API 명세대로 작동해야 합니다.
- 지원자가 README에 제공한 Docker 빌드 및 실행 명령어대로 작동해야 합니다.
- 기능의 모든 요구사항을 구현하지 못해도 평가합니다. (단, 구현 우선순위는 평가에 포함합니다.)

## 기능
1. [바이낸스 API](https://developers.binance.com/docs/binance-spot-api-docs/web-socket-streams#individual-symbol-book-ticker-streams
)를 활용하여 임의의 종목 10개에 대해 최신 시세 정보를 조회
    - 웹소켓의 연결이 종료되는 경우 자동으로 재연결되어야 함
2. (1에서 조회한 데이터에 대해) 클라이언트가 요청한 종목별로 가장 최신 시세 정보 n개를 조회할 수 있는 HTTP API를 제공하며 다음 조건을 만족
    - 조회할 수 없는 종목이 클라이언트의 요청에 포함된 경우 오류 반환
    - n의 최댓값을 임의로 정하고 n을 초과하는 요청인 경우 오류 반환
    - (1에서 조회한 데이터에 대해) n개를 조회할 수 있는 종목만 응답에 포함
    - 응답에 포함된 데이터는 다음 구조를 따름 (단, 시세 객체 구조는 자유)
    ```json
    {
        "price1": [
            {
                "symbol": "BTCUSD",
                ...
            },
            ...
        ],
        ...
        "price{n}": [
            {
                "symbol": "ETHUSD",
                ...
            },
            ...
        ]
    }
    ```
    - API 명세는 REAMDE에 작성
3. (1에서 조회한 데이터에 대해) 클라이언트가 요청한 종목별로 가장 최신 시세 정보를 제공하는 SSE API를 제공하며 다음 조건을 만족
    - 조회할 수 없는 종목이 클라이언트의 요청에 포함된 경우 오류 반환
    - (1에서 조회한 데이터에 대해) 업데이트가 발생할 때마다 업데이트된 내용 별로 응답 전송
    - 응답 형태는 자유이나 프론트엔드 개발자가 응답 받은 시세를 클라이언트 어플리케이션에서 사용한다는 가정 하에 설계
    - API 명세는 REAMDE에 작성
4. Docker Image로 빌드 및 실행
    - 외부 인프라 또는 클라우드에 배포될 어플리케이션 이미지라 가정
    - 빌드 및 실행 방법을 README에 작성
    - 평가자는 REAMDE 매뉴얼대로 실행

## 평가

- 기능 구현 정도

- AI 어시스턴트 활용 능력 (활용 시 REAMDE 등에 활용 기술, 활용 용도 등 기재)
